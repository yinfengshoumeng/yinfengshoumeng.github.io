hexo+gitee搭建博客

笔记
分享技术
发布日期:   2020-04-22

搭建一个博客或个人小屋并不难，难的是坚持写博客记录，维护小屋.这篇文章是教大家搭建一个个人小屋。

那么开始把

Hexo + Gitee 部署自己的个人博客

目前有一些比较火的一些博客框架： Hexo、jekyll、Solo、Halo 、gohugo、VuePress、wordpress 等 ，这些都是开源的静态博客框架，能快速搭建好一个博客站，
用到的知识点不多，需要懂一些前端知识就够了。

准备

  • 安装node.js 环境
    因为Hexo是基于node.js的框架。就像java需要jdk一样。(虽然我已经很久没写过java代码了QwQ)
  • node下载地址：http://nodejs.cn/download/ 。一直点击下一步，傻瓜式安装就可以了。
  • 关于需要用到的npm，安装好node.js会自带npm。
  • 检查安装及配置:
    node -v
npm  -v
  • npm太慢了，建议安装淘宝镜像
    npm install -g cnpm --registry=https://registry.npm.taobao.org
    这样就能使用cnpm了
  • 安装 Git
    学程序的git是必会知识哦，当然，现在有托管工具使用更方便,不用敲代码了QWQ(tortoiseGit:我称其为老乌龟~)
    这篇写的不错哦：https://blog.csdn.net/qq_32370913/article/details/105597326
  • Hexo 官网，安装hexo
    https://hexo.io/zh-cn/
    # 全局安装
npm install hexo-cli -g
# 查看hexo版本
hexo -v

    开始

  • 创建一个本地博客
    blog：你的目录名称
    hexo init blog
  • 进入目录，启动项目 
    # 进入目录
cd blog
# 安装hexo依赖
npm i 
# 启动blog项目
hexo s
    npm太慢了，建议使用 cnpm i
    见证奇迹的时候来了，访问 http://localhost:4000/ 便可看到 hexo 默认的页面和一篇“Hello World”默认生成的文章（从我敲java的时候就Hello World，就不能换个么QwQ）

完成

到此，hexo项目搭建完成

  • 项目主要目录
    .
├── .deploy       # 需要部署的文件
├── node_modules  # 项目所有的依赖包和插件
├── public        # 生成的静态网页文件
├── scaffolds     # 文章模板
├── source        # 博客正文和其他源文件等都应该放在这里
|   ├── _drafts   # 草稿
|   └── _posts    # 文章
├── themes        # 主题
├── _config.yml   # 全局配置文件
└── package.json  # 项目依赖信息
  • 新建一篇自己blog文章
    hexo new "我的第一篇Hexo blog文章"
    然后编辑内容，重启一下站点。第一篇文章完成！（Hexo框架文章中的图片只支持外链的形式，我们可以自己在码云或其他地方部署一个自己的图床）

    部署

    将项目部署到码云
  • 首先你得取创建一个gitee账号呀
    https://gitee.com/
  • 在码云新建一个仓库，注意：仓库名称得和路径一样哦。
  • 选择开源！
  • 选择语言 javaScript
  • 在项目根目录下安装git部署插件
    cnpm install hexo-deployer-git --save
  • 修改项目配置文件：_config.yml 找到关键字deploy
    deploy:
type: git
repo: 仓库URL地址
branch: master
  • 使用命令 hexo d 将项目部署到 gitee 远程仓库
  • 刷新gitee的仓库页，你会看到你提交的代码
  • 开启 Gitee Pages 静态网页托管服务， 点击启动
  • 之后使用提供的网址便是自己博客项目的网站，当时每次重新上传代码到gitee时，都需要点击Gitee Pages更新按钮重启page服务（麻烦github就不会）
  • github和gitee一样哦，不过在国外么……
